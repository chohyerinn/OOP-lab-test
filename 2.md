세미 오픈북이면 문법 자체보다 **UML을 보고 구조를 빨리 잡는 능력**을 볼 가능성이 큽니다.  
아래 6개를 우선순위로 보고 준비하세요.

## 1순위. Static Shared Variable

Likely problem:

> Implement `RoomOccupancy`. Each room has its own people count, but all rooms share one total count.

UML keywords:

```text
- int peopleInRoom
- static int totalNumber
+ void addOneToRoom()
+ void removeOneFromRoom()
+ static int getTotal()
```

Must-memorize pattern:

```java
private int peopleInRoom;
private static int totalNumber = 0;

public void addOneToRoom() {
    peopleInRoom++;
    totalNumber++;
}

public void removeOneFromRoom() {
    if (peopleInRoom > 0) {
        peopleInRoom--;
        totalNumber--;
    }
}

public static int getTotal() {
    return totalNumber;
}
```

Important:

- `static` means shared by all objects.
- Call static methods using the class name: `RoomOccupancy.getTotal()`.
- Call instance methods using an object: `roomA.addOneToRoom()`.

## 2순위. 2D Array Static Method

Likely problem:

> Write a static method `findFigure(double[][] picture, double threshold)` that returns a new 2D array.

UML keywords:

```text
+ static double[][] findFigure(double[][] picture, double threshold)
- static double average(double[][] picture)
```

Must-memorize pattern:

```java
public static double[][] findFigure(double[][] picture, double threshold) {
    double sum = 0;
    int count = 0;

    for (int r = 0; r < picture.length; r++) {
        for (int c = 0; c < picture[r].length; c++) {
            sum += picture[r][c];
            count++;
        }
    }

    double average = sum / count;
    double[][] result = new double[picture.length][picture[0].length];

    for (int r = 0; r < picture.length; r++) {
        for (int c = 0; c < picture[r].length; c++) {
            if (picture[r][c] > average * threshold) {
                result[r][c] = 1.0;
            } else {
                result[r][c] = 0.0;
            }
        }
    }

    return result;
}
```

Important:

- 2D array means nested loops.
- Row count: `picture.length`.
- Column count: `picture[r].length`.
- Return a **new** array, not the original array.

## 3순위. Abstract Class + Inheritance

Likely problem:

> Implement abstract class `PayCalculator`, then derive `RegularPay` and `HazardPay`.

UML keywords:

```text
<<abstract>> PayCalculator
- double payRate
+ double computePay(double hours)

RegularPay extends PayCalculator
HazardPay extends PayCalculator
```

Must-memorize pattern:

```java
public abstract class PayCalculator {
    private double payRate;

    public PayCalculator(double payRate) {
        this.payRate = payRate;
    }

    public double computePay(double hours) {
        return payRate * hours;
    }
}
```

```java
public class HazardPay extends PayCalculator {
    public HazardPay(double payRate) {
        super(payRate);
    }

    @Override
    public double computePay(double hours) {
        return super.computePay(hours) * 1.5;
    }
}
```

Important:

- Abstract class can have normal methods.
- You cannot create `new PayCalculator(...)`.
- `super(payRate)` calls the parent constructor.
- `super.computePay(hours)` reuses the parent calculation.

## 4순위. Person-Student Inheritance

Likely problem:

> Implement `Person`, `Student`, and maybe `Undergraduate` using inheritance and overriding.

UML keywords:

```text
Person <|-- Student
Student <|-- Undergraduate
+ void writeOutput()
+ boolean equals(...)
```

Must-memorize pattern:

```java
public class Student extends Person {
    private int studentNumber;

    public Student(String name, int studentNumber) {
        super(name);
        this.studentNumber = studentNumber;
    }

    @Override
    public void writeOutput() {
        super.writeOutput();
        System.out.println("Student Number: " + studentNumber);
    }
}
```

Important:

- Child class uses `extends`.
- Parent constructor call must be first: `super(name);`.
- Overriding means same method name and same parameters.
- A child object is also a parent object:

```java
Person p = new Student("Kim", 20260001);
```

## 5순위. Custom Exception + Try-Catch

Likely problem:

> Define `DivideByZeroException` or `NegativeNumberException` and use it in a method.

UML keywords:

```text
DivideByZeroException extends Exception
+ double divide(double a, double b) throws DivideByZeroException
```

Must-memorize pattern:

```java
public class DivideByZeroException extends Exception {
    public DivideByZeroException() {
        super("Division by zero.");
    }
}
```

```java
public static double divide(double a, double b)
        throws DivideByZeroException {
    if (b == 0) {
        throw new DivideByZeroException();
    }
    return a / b;
}
```

```java
try {
    double result = divide(10, 0);
    System.out.println(result);
} catch (DivideByZeroException e) {
    System.out.println(e.getMessage());
}
```

Important:

- Exception class: `extends Exception`.
- Method warning: `throws ExceptionName`.
- Actually make the error: `throw new ExceptionName();`.
- Handle the error: `catch (ExceptionName e)`.
- `throw` and `throws` are different.

## 6순위. Text File I/O

Likely problem:

> Read from a text file, copy lines, or process CSV-style data.

UML keywords:

```text
+ int copyLines() throws IOException
+ void processFile() throws FileNotFoundException
```

Read file pattern:

```java
try {
    Scanner input = new Scanner(new File("input.txt"));

    while (input.hasNextLine()) {
        String line = input.nextLine();
        System.out.println(line);
    }

    input.close();
} catch (FileNotFoundException e) {
    System.out.println("File not found.");
}
```

Write file pattern:

```java
try {
    PrintWriter output = new PrintWriter("output.txt");
    output.println("Hello");
    output.close();
} catch (FileNotFoundException e) {
    System.out.println("Cannot write file.");
}
```

Important:

- Reading needs `Scanner` and `File`.
- Writing needs `PrintWriter`.
- File code usually needs `try-catch`.
- Remember imports:

```java
import java.io.File;
import java.io.FileNotFoundException;
import java.io.PrintWriter;
import java.util.Scanner;
```

## Final 30-Minute Study Order

1. Memorize static pattern.
2. Memorize 2D array nested loop.
3. Memorize `extends`, `super`, `@Override`.
4. Memorize custom exception: `extends Exception`, `throws`, `throw new`, `catch`.
5. Memorize file read/write patterns.
6. Quickly review the full prediction pack only after these patterns are comfortable.

---

## What Not To Waste Too Much Time On

- Perfect formatting of output.
- Complicated UI or menu design.
- Memorizing every problem from the long prediction pack.
- Very small syntax details that can be checked in semi-open-book materials.

Focus on structure first. Syntax can be fixed if the structure is right.
